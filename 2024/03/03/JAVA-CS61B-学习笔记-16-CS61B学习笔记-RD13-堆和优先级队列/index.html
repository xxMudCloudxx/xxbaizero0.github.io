CS61B学习笔记(十六)-RD13-堆和优先级队列

为什么要使用PQ？

我们学到的最后一个 ADT 是二叉搜索树，它使我们能够高效搜索，只需要 logN 时间。这是因为我们可以在搜索的每一步中剔除一半的元素。如果我们更关心快速找到最小或最大元素而不是快速搜索怎么办？

同时，二叉树中，我们不允许存在两个具有相同的值的情况重新，因此就有了PQ。

优先级队列接口

要理解这个抽象数据类型，请考虑一袋物品。您可以向这个袋子中添加物品，也可以从中移除物品等等。唯一的限制是您只能与这个袋子中的最小的物品进行交互。

使用优先级

我们何时会真正使用或需要这种结构？

下面是一个简单的例子，演示了如何使用优先队列来解决任务调度的问题：

假设有一个任务调度系统，有多个任务需要执行，每个任务都有一个优先级。我们希望按照任务的优先级来执行任务，优先级高的任务优先执行，但是如果有多个优先级相同的任务，我们希望按照它们的到达时间先后来执行。

在这种情况下，可以使用优先队列来管理待执行的任务队列。每次从优先队列中取出优先级最高的任务来执行，如果有多个优先级相同的任务，则按照它们的到达时间先后来执行。

概括

  • 使用这种专门的数据结构可以带来空间/内存方面的好处。

  • 我们目前所知的 ADT 实现并不能为 PQ 操作提供高效的运行时。

    • 在其他结构中，二叉搜索树是最有效的

堆结构

注意:这个堆并不是我们所熟知的堆栈

我们之前已经看到，我们所知的数据结构中对于优先队列操作的运行时最好的是二叉搜索树。通过修改其结构和约束条件，我们可以进一步提高这些操作的运行时和效率。

我们将定义我们的二叉最小堆为完全堆，并遵守最小堆性质：

  • 最小堆： 每个节点都小于或等于其两个子节点。
  • 完全堆： 底层（如果有）仅缺少项，所有节点都尽可能靠左。

正如我们在上面的图中所看到的，绿色的堆是有效的，而红色的堆则不是。后两个不是有效的，因为它们违反了我们上面定义的至少一个属性。

现在让我们考虑这种结构如何适用于我们在前一章中描述的抽象数据类型。我们将通过分析我们期望的操作来做到这一点。

堆操作

对于优先队列 ADT，我们关心的三种方法是 add、getSmallest 和 removeSmallest。我们将从概念上描述如何根据我们给定的堆模式实现这些方法。

  • add：

    暂时添加到堆的末尾。向上游到适当位置。

    • 上游涉及交换节点，如果子节点小于父节点。

  • getSmallest： 返回堆的根（根据我们的最小堆属性，这是最小值）。

  • removeSmallest：

    将堆中的最后一个项目与根交换。向下沉到适当的位置。

    • 下沉涉及交换节点，如果父节点大于子节点。与最小子节点交换以保持最小堆属性。
