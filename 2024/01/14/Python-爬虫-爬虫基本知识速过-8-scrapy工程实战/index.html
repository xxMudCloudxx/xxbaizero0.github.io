第二步：分析网页

创建完项目后，我们来做一些分析、准备工作。

确定爬取内容

打开科技人物板块网页可以看到，页面主体部分是板块下新闻列表，每篇新闻都是来自各个新闻资讯平台的文章，以摘要的形式展现。

打开一篇文章，可以看到它有 标题、来源、作者、时间 和 正文 这五项内容。这些是我们对于每篇文章都要爬取的部分。

确定了要爬取的数据后，就要定义数据格式啦。我们找到 items.py 文件，在自动创建的 Article 类里设置 title、time、source、author 和 content 五个变量，分别对应着上述提到的内容。

确定好需要爬取的内容，在正式编写爬虫前，我们还有一些准备工作需要完成。一是分析环球网新闻列表页是如何加载这些文章摘要的。二是分析如何提取文章页中的各项元素。

分析新闻列表页

首先打开环球网科技人物板块页面 https://tech.huanqiu.com/per ，并打开开发者工具，选择 network 面板，并勾选 Fetch/XHR 按钮，方便我们后续的观察。

尝试刷新、滚动页面，分析每项操作后网页的变化以及产生的请求：

可以发现这是一个动态加载的页面，可以不断地向下滚动，溯源到之前的新闻。而每次向下滚动到一定程度时，可以在开发者工具看到新出现了一个长得像这样的请求：

这表明它应该就是环球网用来动态加载文章的 API。这个 URL 包含三个参数，node、offset 和 limit，滚动时 node 和 limit 的值不变，而 offset 随着滚动从 0 变成 20、40、60……

node 值没有变化，所以我们不用去管它。而一直在变化的 offset，它和 limit 是经常出没在这种动态加载页面中的黄金组合。把板块下所有文章看作一个列表的话，offset 相当于 索引。而 limit 表示从 offset 位置开始，最多可以获取多少条数据。

上面的 URL 中 offset 值为 0，limit 值为 20，表示从所有新闻中的第一篇文章开始，获取 20 篇文章信息。尝试改变一下 offset 的值，发现当 offset 值为 20 时，加载出了第 21～40 篇文章；offset 值为 40 时，加载出了第 41～60 篇文章。所以我们只用在原 URL 基础上改变 offset 的值，就能不断地加载文章了。

在编程时，我们可以设置一个全局变量 LIST_URL 来保存它。其中每次都会改动的部分，也就是 offset 参数的值，以 {} 的形式空了出来。这样一来，我们只要通过字符串的 format() 方法传入不同的值，就能访问到不同的链接了。

分析完链接，我们再来看看它的响应内容：

可以看到响应内容是个 JSON 格式数据，以列表的形式保存着一篇篇文章的摘要等信息。仔细观察第一篇文章中包含的信息，可以发现有个 aid，值为 44PjZH52nz0。它跟文章页面之间有什么样的关联呢？下面，我们来分析一下文章页，一探究竟。
