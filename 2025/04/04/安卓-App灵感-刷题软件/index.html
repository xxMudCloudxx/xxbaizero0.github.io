App灵感-刷题软件

需求分析

  • 应用能够从包含答案的 Word 或 PDF 题库中逐题提取题目信息。

界面分解

标签栏

  • 首页
  • 练习

首页页面

  • 题库
  • 创建错题
  • 全部错题
  • 上传框

练习页面

题库列表

做题页面

Bar

  • 答题模式
  • 背题模式

做题模式

  • 显示题目类型、具体题目内容、选项。
  • 底部标签栏包括：
    • 收藏题目功能
    • 正确、错误、不确定题目数量统计
    • 题目列表快速导航按钮

背题模式

在做题模式的基础上增加解析和笔记区

错题页面/收藏页面

  • 提供导出功能，允许用户导出错题或收藏。
  • 支持错题练习和收藏练习，使用户能够有针对性地练习自己的弱点或感兴趣的题目。

用 React Native 来实现这个刷题 App 是个很好的选择，它既能跨平台，也有大量现成组件库可用。根据你目前的需求分析和功能结构，我们可以设计一个清晰、可扩展的项目结构。

项目结构

核心模块解释

1. navigation/

使用 react-navigation 管理：

  • Tab 导航（首页、练习）
  • Stack 导航（进入题库 → 做题页面 → 错题/收藏）

2. screens/

每个页面都拆一个目录，便于维护样式、逻辑与组件。

3. services/fileParser.js

这是一个重点模块：

  • 将 Word/PDF 题库解析成结构化 JSON（推荐用 react-native-pdf + 后端辅助解析，或前端用 pdf-parse）

4. context/QuizContext.js

用 React Context + useReducer 管理全局数据：

  • 当前题库、答题进度、错题/收藏状态等。

🚀 技术推荐

功能 技术栈
路由 react-navigation
状态管理 React Context or Zustand
PDF解析 后端解析（Node + pdf-parse），前端处理结果
文件上传 expo-document-picker or react-native-document-picker
本地存储 AsyncStorage or react-native-mmkv
题目显示 FlatList + 自定义题目组件
主题样式 styled-components or tailwind-rn

✅ TODO优先级（开发节奏建议）

  1. 页面结构与导航
  2. 题库上传 + 题目解析（可先手动录入 mock 数据调试）
  3. 做题逻辑（切题、答题记录、错题统计）
  4. 收藏/错题练习模块
  5. 背题模式 + 解析区
  6. 数据导出（导出错题 PDF/JSON）
  7. 优化 UI/UX、加入动画、主题切换等
